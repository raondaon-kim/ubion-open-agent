## 왜 시 에이전트인가

Phase 0의 후보 3개 중 본 시나리오를 고른 이유 (사용자 결정 2026-05-13):

1. **창의/탐색 축의 자기진화 가치가 가장 가시적** — 같은 사용자가 같은 에이전트와 반복 작업할수록 *취향과 결*이 누적되는 것을 본인이 즉시 체감 가능
2. **memory의 가치 검증에 직결** — 사용자의 취향, 자주 쓰는 표현, 피드백 패턴을 기억하는지가 시 품질에 즉시 반영됨
3. **skill 자동 생성의 자연스러운 트리거** — "이 사용자는 비유를 좋아한다", "수정 시 이런 단계를 거친다" 같은 패턴이 skill로 응축되기 좋음
4. **검증 비용이 낮음** — PDF 파이프라인이나 Genlearn 작업처럼 외부 시스템 연결이 필요 없음. 채팅 + memory + skill 만으로 완결

## 시나리오 정의

### 사용자 행동 (1주 동안)

- 매일 최소 1세션, 5~10턴 대화
- 작업 종류 (자유 혼합, 사용자 재량):
  - 새 시 한 편 창작 요청
  - 기존 시 수정/다듬기 요청
  - 시상 brainstorming
  - 특정 주제/형식/감정 요청
  - 결과물에 대한 피드백 ("이 표현은 좋다", "이 부분은 진부하다")
- 사용자는 명시적으로 skill을 만들라고 요청하지 않음 — 에이전트가 자기 판단으로 skill 만들기를 기대

### 측정 항목

매일 끝에 다음을 기록 (사람이 채움):

| 항목 | 측정 방법 |
|------|---------|
| 세션 수 / 턴 수 | 카운트 |
| 결과물을 *실제로* 사용했는가 | y/n |
| 사람이 추가로 손본 양 | 0~10 척도 (0 = 그대로 사용, 10 = 거의 새로 씀) |
| 새 skill 자동 생성 수 | `~/.ubion-agent/skills/`에서 카운트 |
| 새 skill 재사용 횟수 | `.usage.json`의 `use_count` 증분 |
| memory에 누적된 사용자 특성 메모 수 | `MEMORY.md` 또는 memory store에서 카운트 |
| 다음 세션에서 이전 결정/취향이 자동으로 반영된 사례 | 메모 (정성) |
| 잘못 기억하거나 부적절한 skill 생성 사례 | 메모 (정성) |
| 세션당 LLM 비용 | usage_pricing 기록 또는 Anthropic 콘솔 |

### 게이트 — Phase 1 GREEN 조건

다음을 모두 만족해야 Phase 1 GREEN:

- [ ] 1주 동안 매일 사용 (최소 5일)
- [ ] **자동 생성된 skill 최소 3개** 이며, 사람이 봤을 때 *재사용 가능*한 수준
- [ ] **이전 세션의 결정/취향이 다음 세션에 반영된 명확한 사례 2회 이상** (memory가 동작한다는 증거)
- [ ] **같은 종류 작업 반복 시 "사람이 추가로 손본 양"의 감소 추세** (3회 이상 데이터)
- [ ] 1주 누적 LLM 비용 ≤ 사전 상한 (Phase 1 시작 시 사용자가 정함)
- [ ] 에이전트가 *부적절한 skill을 생성하거나 잘못 기억해서 작업을 망친 사례*가 일주일 동안 2회 이하

### 게이트 실패 시

- 🟡 부분 실패 (skill은 만들었으나 memory 약함 등) → 약한 단위에 대해 Phase 1.5로 보완 작업 정의
- 🔴 명확한 실패 (skill 0개, memory 동작 안 함, 비용 폭주, 결과물 품질 낮음) → 단위별 회귀, 또는 자기진화 가치 자체 재검토

## 시나리오를 위해 필요한 기술 단위 (참고용)

본 시나리오 검증을 위해 필수로 동작해야 할 단위 (`phase-1-todo.md` 기준):

- 단위 1, 2 ✅ — 메시지 → 응답 가능
- 단위 4 — system prompt에 skill 인덱스 + memory 인덱스 주입 (없으면 에이전트가 자기 skill을 모름)
- 단위 6 — memory manager (없으면 다음 세션 기억 불가)
- 단위 7 — skill loading (skill_view 사용)
- 단위 8 부분 — skill_manage (action=create) tool (없으면 에이전트가 새 skill 만들 수 없음)
- 단위 9 — session search (없으면 "이전에 비슷한 작업 했던 거 찾아줘" 안 됨)
- 단위 11 — git auto-commit (안전망. rollback 가능해야 함)

단위 3, 5, 10은 안 해도 시 에이전트는 동작 가능. 단, SPEC §9는 (A) 11단위 완료 후 (B) 진입이라 11단위는 다 끝내고 들어옴.

## 실행 절차 (Phase 1 종료 시점)

1. ax 센터가 `~/.ubion-agent/` 디렉토리 초기화 + SOUL.md/USER.md 작성
   - SOUL.md: 시 에이전트의 정체성 (예: "사용자의 취향을 학습하면서 함께 시를 쓰는 동반자")
   - USER.md: ax 센터 본인 정보 + 선호 시 스타일
2. **시나리오 시작 직전 수동 git snapshot** (단위 11 보류 결정 2026-05-13 반영):
   ```
   cd ~/.ubion-agent
   git init
   git add .
   git commit -m "scenario-start"
   ```
   사고 발생 시 `git reset --hard scenario-start` 로 즉시 rollback.
3. `python -m engine.server` (단위 10) 기동. 사용자가 `UBION_WORKSPACE` 를 시 작업 폴더로 설정 (예: `set UBION_WORKSPACE=D:\poems\`)
4. 매일 세션 진행 + 측정 항목 기록
5. **매일 끝에 수동 commit**: `git add . && git commit -m "day-N"` — 어느 시점에 어떤 skill/memory 가 생겼는지 추적
6. 1주 후 회고 작성 → `research/phase-1-retrospective.md`. `git log` 로 시간순 변화 첨부
7. 게이트 판정 → Phase 2 진입 또는 보완 작업
8. 회고에서 *자동화 가치가 입증된 트리거 패턴* 만 단위 11 로 진행
